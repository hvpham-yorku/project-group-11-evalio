Story 1: ITR2-2 — Persistent Multi-Course Planning System

Test Case 1

Goal: verify a user can create and keep multiple courses separately.

Steps 1. Log in and create Course A. 2. Add a different Course B. 3.
Open the course list and switch between both courses. 4. Refresh the
page and open the list again.

Expected Result - Both courses remain saved. - Switching courses updates
the active workspace. - Refresh does not remove either course.

Actual Result - Multi-course creation and retrieval worked correctly. -
Course switching stayed scoped to the selected course. - Data remained
available after repeated fetches.

Status: Pass

Test Case 2

Goal: verify saved scenarios stay isolated per course.

Steps 1. Open Course A and save a scenario. 2. Switch to Course B. 3.
Check the saved scenario list in Course B. 4. Return to Course A and
open the saved scenario again.

Expected Result - Course B does not show Course A scenarios. - Course A
still keeps its own saved scenario.

Actual Result - Scenario isolation worked correctly across courses. -
Scenario CRUD lifecycle passed under course scoping.

Status: Pass

Test Case 3

Goal: verify cross-user and cross-course protection on update and
delete.

Steps 1. Create courses under one authenticated user. 2. Attempt update
and delete operations through a mismatched user/course scope. 3. Recheck
the original user’s courses.

Expected Result - Invalid cross-scope actions are rejected. - Original
course data remains unchanged.

Actual Result - Invalid scoped update/delete attempts returned 404. -
Original course state remained unchanged.

Status: Pass

Story 2: ITR2-3 — Universal GPA Converter

Test Case 1

Goal: verify exact GPA scale conversion works for supported scales.

Steps 1. Open the GPA converter. 2. Set current GPA to 8.2 on a 9.0
scale. 3. Convert to the 4.0 scale.

Expected Result - The converted GPA is returned correctly. - The formula
and normalized percent are shown consistently.

Actual Result - Conversion returned 3.6444. - Conversion flow behaved
consistently with the API contract.

Status: Pass

Test Case 2

Goal: verify invalid GPA values are rejected.

Steps 1. Submit a GPA higher than the source scale. 2. Retry with a
valid value.

Expected Result - Invalid value is rejected with a clear error. - Valid
value works normally.

Actual Result - Invalid current GPA was rejected with 400. - Valid
conversion continued to work after correction.

Status: Pass

Test Case 3

Goal: verify weighted cGPA calculation works with multiple courses.

Steps 1. Submit one course with 85% and 3 credits. 2. Submit one course
with 70% and 6 credits. 3. Request cumulative GPA.

Expected Result - cGPA is weighted by credits. - Returned formula
matches the weighted inputs.

Actual Result - Weighted cGPA returned 6.67 on the requested scale. -
The formula matched the weighted calculation.

Status: Pass
